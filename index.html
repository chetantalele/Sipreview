Summer Internship Project (SIP) Review

at Celebal Technologies

By: Chetan Talele | June–July 2025

Project Overview

The final project was an Intelligent Flight Ticket Booking Bot that allows users to search, compare, and book flights using a chat interface. It streamlines the booking process by integrating real-time flight data APIs and responds to user queries via natural text inputs.

Week-wise Learning Plan

Week 1 - Introduction to Node.js

Key Topics: Node.js Architecture (V8 Engine, Event-Driven, Non-Blocking I/O), The Node.js Ecosystem and Modules, Understanding package.json and Node.js REPL.

Assignment: Environment Setup & First Application. To install Node.js and NPM, and to create and run a basic "Hello World" server.

Outcome: Successfully configured the development environment, which was foundational for all subsequent tasks.

Week 2 - Core Modules & NPM

Key Topics: Utilizing core modules like `fs` (File System), `http` (HTTP server), `path`, and `util`. Deep diving into NPM for package management and scripting.

Assignment: File Management Tool. Built a command-line tool to perform file operations using Node.js core modules.

Outcome: Gained practical experience in handling asynchronous file operations and managing project dependencies with NPM.

Week 3 - Asynchronous Programming

Key Topics: In-depth study of the Node.js Event Loop, callback hell, Promises, and the modern `async/await` syntax for writing clean asynchronous code.

Assignment: Promise-based file operations. Refactored previous assignments to use Promises and async/await, improving code readability and error handling.

Outcome: Developed a solid understanding of Node.js's asynchronous nature, which is crucial for building performant web applications.

Week 4 - Express.js Basics & Middleware

Key Topics: Setting up an Express.js server, defining routes, using URL parameters, and understanding how middleware works for request processing.

Assignment: Express web server. Created a multi-route web server that served static files and handled basic API requests.

Outcome: Gained experience in building a structured web server and utilizing Express's core features for routing and serving content.

Week 5 - MongoDB CRUD & Schema Design

Key Topics: Integrating MongoDB with Node.js, Mongoose ODM for schema design, and implementing Create, Read, Update, and Delete (CRUD) operations.

Assignment: CRUD app with MongoDB. Developed a simple REST API that connected to a MongoDB database to manage a collection of data.

Outcome: Successfully linked the Node.js backend with a NoSQL database, learning how to persist and retrieve data effectively.

Week 6 - REST APIs & Error Handling

Key Topics: Designing RESTful API endpoints, handling various HTTP methods (GET, POST, PUT, DELETE), and implementing a robust error-handling middleware.

Assignment: RESTful API development. Built a clean, testable API with proper routing and a global error-handling mechanism.

Outcome: Understood the principles of REST and created a well-structured API ready for client-side consumption.

Week 7 - JWT Authentication & Security

Key Topics: Implementing JSON Web Tokens (JWT) for stateless authentication, using `bcrypt` for secure password hashing, and adding middleware to protect routes.

Assignment: JWT Auth system. Developed user login and registration endpoints, securing private routes with JWT tokens.

Outcome: Gained crucial knowledge in securing API endpoints and user data, a fundamental skill for backend development.

Week 8 - Advanced Express & Project Enhancement

Key Topics: Advanced Express patterns, file uploads with `multer`, and reviewing the entire project structure for scalability and performance optimizations.

Assignment: Enhance Express app. Integrated new features and refactored code to finalize the project for the review.

Outcome: Completed the final project and gained a comprehensive understanding of the full backend development lifecycle.

Project Details

🎯 Internship Objective

  • Gain hands-on backend development experience.
  • Learn Node.js, Express.js, MongoDB, and RESTful API development.
  • Understand backend architecture, security, and deployment practices.
  • Develop a full-stack intelligent chatbot for flight booking.

🛠️ Technology Stack

  • Backend: Node.js, Express.js
  • Database: MongoDB, Mongoose
  • Authentication: JWT, bcrypt
  • Testing: Postman
  • Version Control: Git + GitHub

System Architecture

  • User sends input via chat interface.
  • Backend processes request using Express.js.
  • Server fetches flight info from third-party API.
  • Database stores booking and user info securely.
  • Chatbot returns best options to the user in real time.

Key Features

  • User registration & JWT login.
  • Real-time flight search and booking.
  • Modular API with error handling & validation.
  • Clean folder structure and reusable routes.

Challenges & Solutions

  • API Complexity: Used mock data & tested with Postman.
  • Async Handling: Adopted async/await patterns with try/catch.
  • JWT Setup: Implemented secure tokens with bcrypt & headers.
  • Modularity: Separated logic using routers/controllers.
